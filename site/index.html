TUI-first Rust coding agent

Repository work, kept inside one terminal interface.

Sigil brings chat, tool calls, approvals, diffs, diagnostics, planning, and session recovery into a terminal product built for real coding sessions.

Set up

Workflow

Designed for tool-backed coding sessions.

Chat-first TUI

Use the composer for ordinary coding work, inspect streamed output, and keep session, usage, permissions, agents, and LSP state visible.

Durable planning

Use /plan for multi-step work. Planner, executor, and optional subagent roles write recoverable task state.

Diff-aware tools

File writes, edits, deletes, shell commands, MCP calls, and code-intel edits run through structured tool and approval surfaces.

Session recovery

Append-only JSONL logs let Sigil restore context without silently replaying interrupted tool execution.

Safety model

Auditable control before risky actions.

  • Permission decisions for write, execute, network, MCP, and external data access.
  • Bounded previews and diffs before file-changing tools are approved.
  • Provider-specific behavior stays in provider crates, not in the generic kernel.
  • Tool approval and execution lifecycle are durable control records.
  • Interrupted tools become interrupted results on restore, not hidden retries.
  • MCP servers run under explicit trust, approval, and secret-egress policy.
