TUI-first Rust coding agent

把真实仓库开发收进一个终端界面。

Sigil 把对话、durable task、child agent、审批、diff、诊断、verification 和 session 恢复放进面向 coding session 的终端产品里。

开始使用 查看源码
sigil 
> /task audit the release docs
planner  写入 durable steps
worker   完成只读 review
tool     edit_file preview ready
verify   pages check, docs links
status   alpha release published

快速开始

安装 preview release，并在工作区里启动。

1

安装

npm install -g @sigil-ai/sigil@alpha
# 或：brew install JimmyDaddy/sigil/sigil-ai
# 或：cargo install --git https://github.com/JimmyDaddy/sigil --tag v0.0.1-alpha.1 --locked sigil
2

启动

cd /path/to/your/project
sigil
3

配置

找不到可用配置时，Sigil 会打开 Quick Setup。确认 workspace、选择 provider/model，并填写认证信息。v0.0.1-alpha.1 是 early preview。

工作流

为可审计的 coding session 设计。

Workspace TUI

直接在 composer 输入任务，查看流式输出，并保持 session、usage、permissions、agents、LSP 和 controls 可见。

Durable tasks and agents

用 /task 处理多步骤任务，用 /plan 做只读规划，用 child agent 承接有边界的委派工作。

Approval and verification

文件编辑、命令、MCP 调用、LSP edit 和配置化检查都经过结构化 preview、receipt 和 review surface。

Configurable runtime

Provider、permission、terminal、appearance、code intelligence、MCP、plugins、skills 和 agents 都收敛在 TUI-first 工作流里。

安全模型

高风险动作前保留可审计控制权。

  • 写入、执行、网络、MCP、LSP edit 和外部目录访问都经过 permission decision。
  • 高风险动作完成前保留有边界的 preview、diff 和 verification evidence。
  • Provider 专项行为留在 provider crate，不污染通用 kernel。
  • Tool、task、agent、approval 和 execution lifecycle 都是 durable control records。
  • 中断工具在恢复时变成 interrupted result，而不是隐藏重试。
  • MCP server、plugin、workspace agent 和外部数据都受显式 trust policy 约束。
